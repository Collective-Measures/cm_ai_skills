DO

  • Wrap in ResponsiveContainer
  • Use Practix chart palette
  • Format tooltip values with formatters
  • Put titles in card header, not chart
  • Use tabular-nums on rendered values

DON'T

  • Use default Recharts colors
  • Add 3D effects or bar gradients
  • Use pie charts for 5+ segments
  • Use Chart.js, D3, or Plotly
  • Leave default Legend/Tooltip styling

Quality Checklist

Run through before sharing any prototype for review

Layout & Structure

Dark sidebar present — w-64, bg-slate-800, sticky, full height
Sidebar has logo area with tool name subtitle
Header bar present — h-16, bg-slate-800
Content area — bg-slate-50 with p-6 padding
No top-nav-only layout — sidebar is mandatory

Colors

Using slate scale — NOT gray, NOT zinc
Primary accent is blue-600
Status colors: green/red/amber/yellow/blue
White cards on slate-50 background

Typography

No custom fonts — system font stack only
tabular-nums on all numeric values
Body text is text-sm, not text-base

Components

Cards use rounded-lg border border-slate-200 shadow-sm
Buttons: blue-600 primary, border secondary
Inputs have focus:ring-2 focus:ring-blue-500
Loading states use animate-pulse skeletons

Data & Charts

Using Recharts — not Chart.js, D3, or Plotly
Chart colors match Practix palette
Numbers formatted with fmtCurrency/fmtCompact/fmtPct
Null/NaN values show "--"

Icons & Code

Using Lucide React — not Font Awesome or Heroicons
No inline styles — all Tailwind utilities
No CSS files per component
No console.log left in code
